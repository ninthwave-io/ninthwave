severity: medium
description: |
  `nw pr-create` (wrapping `gh pr create`) failed with
  `head branch "main" is the same as base branch "main", cannot create a pull request`
  when invoked from inside a worktree on branch `ninthwave/M-STACK-1` (verified via
  `git rev-parse --abbrev-ref HEAD`). The branch was already pushed and tracking
  `origin/ninthwave/M-STACK-1`. Passing `--head ninthwave/M-STACK-1` explicitly
  worked. Suggests `gh pr create` is resolving the head branch incorrectly inside
  the worktree (or `nw pr-create` is suppressing/overriding `--head`). The implementer
  prompt's `nw pr-create` examples do not include `--head`, so workers will hit this
  the same way unless either (a) the underlying detection is fixed, or (b) the
  prompt guidance recommends `--head $(git rev-parse --abbrev-ref HEAD)` (or
  `--head ninthwave/YOUR_WORK_ITEM_ID`) for worker invocations.
