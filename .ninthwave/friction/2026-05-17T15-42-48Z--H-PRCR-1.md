item: H-PRCR-1
date: 2026-05-17T00:00:00Z
severity: medium
description: |
  Hit the exact bug I was fixing while creating the PR for the fix. The
  system-installed `nw pr-create` (PATH-resolved) failed with
  "head branch 'main' is the same as base branch 'main'" when invoked
  from the worktree. Workaround: ran `bun run core/cli.ts pr-create ...`
  so the worktree-local patched code path was used. Worth noting: a
  worker fixing a tool's own bug cannot easily eat its own dogfood
  unless we route `nw` invocations through the worktree's source rather
  than the global binary. Could be addressed by a wrapper that prefers
  `bun run core/cli.ts` when run from inside a ninthwave worktree, or by
  documenting this footgun for self-hosting work items.
