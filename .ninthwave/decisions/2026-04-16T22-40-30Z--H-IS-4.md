item: H-IS-4
date: 2026-04-16T00:00:00Z
summary: Repurposed `p` key from modal pause to acceptingWork toggle.
context: The spec asked for `p` to toggle acceptingWork (mnemonic "pause intake"), but `p` already toggled the existing modal "paused" overlay. The two concepts are semantically distinct -- the modal pause freezes the TUI behind a centered overlay, while acceptingWork keeps the dashboard fully active but stops new launches.
decision: `p` now toggles `acceptingWork`. The modal paused overlay is still reachable via Escape from the base dashboard (Escape was already bound to pause in that case), and Escape resumes from inside the modal (swapped the in-overlay `p` resume binding to Escape). The paused-overlay footer hint was updated to "esc resume" accordingly.
rationale: (1) The work item explicitly specifies `p` with the "pause intake" mnemonic, so using a different key would break the stated UX. (2) acceptingWork is the more useful daily pause semantic (you can still see and interact with the dashboard while draining), so `p` is better spent on it. (3) Keeping the modal pause reachable via Escape preserves the existing capability without requiring a separate work item for the behaviour change. (4) Users who press `p` while inside the modal will resume via Escape -- documented in the overlay footer.
